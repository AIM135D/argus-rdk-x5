# 硬件与接线

## 1. 组成

- RDK X5；
- USB/MIPI 固定广角摄像头；
- ESP32；
- 水平、俯仰两个舵机；
- 蜂鸣器及其驱动电路；
- 可选灯光执行器；
- 独立舵机电源、共地和机械限位。

## 2. 数据关系

摄像头只连接 RDK X5。RDK X5 完成图像处理和决策，再通过 USB 串口向 ESP32
发送目标方向和报警等级。ESP32 负责实时 PWM、滤波、超时回中和蜂鸣节奏。

## 3. 当前公开固件引脚

| 功能 | ESP32 GPIO | 说明 |
|---|---:|---|
| 水平舵机 | 25 | 固件常量 `PAN_PIN` |
| 俯仰舵机 | 26 | 固件常量 `TILT_PIN` |
| 蜂鸣器 | 27 | 默认低电平有效 |
| 灯光 | 未绑定 | 协议保留 `light` 字段，需现场选择 GPIO 和驱动 |

源码位置：

```text
firmware/esp32/active_warning_controller/active_warning_controller.ino
```

## 4. 供电

- 不要用 RDK X5 或 ESP32 的 3.3V 引脚直接给舵机供电；
- 舵机使用独立、满足峰值电流的电源；
- RDK X5、ESP32、舵机电源必须共地；
- 感性负载、蜂鸣器和灯光应使用合适的三极管/MOSFET、续流和保护电路；
- 上电前设置机械限位，确保 90° 中位不会碰撞。

## 5. 串口

默认：

```text
设备：/dev/ttyUSB0（可配置）
波特率：115200
格式：UTF-8/ASCII 行，以 \n 结束
```

不要把串口权限永久开放给所有用户。优先使用 udev 规则和用户组。

## 6. 首次通电顺序

1. 保持 ARGUS 所有硬件开关为 false；
2. ESP32 单独上电，确认串口输出 `READY`；
3. 不连接舵机负载，先验证 `C`、`L` 和 ACK；
4. 连接独立舵机电源并共地；
5. 逐个舵机验证中位、方向和机械范围；
6. 完成 3×3 标定；
7. 最后启用蜂鸣器；
8. 灯光必须在确定 GPIO、驱动和失效状态后单独接入。

## 7. 失效处理

ESP32 超时未收到目标时会进入丢失状态并停止报警。此机制只是软件回退，
不能代替急停、机械限位、继电器安全回路或人工监护。
