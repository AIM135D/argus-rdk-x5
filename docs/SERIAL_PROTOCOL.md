# RDK X5 ↔ ESP32 串口协议

所有命令为单行 CSV，以换行结束。默认 115200 baud。

## 1. 推荐角度协议 `A`

```text
A,seq,zone_id,pan,tilt,score,light,beep
```

示例：

```text
A,42,1,96,78,0.873,1,2
```

- `seq`：递增序号；
- `zone_id`：危险区域 ID；
- `pan/tilt`：0–180°，RDK 侧标定后给出；
- `score`：0–1 风险分数；
- `light`：0/1 逻辑字段；
- `beep`：0–3 报警等级。

`A` 模式适合在 RDK 侧使用稀疏网格和区域覆盖完成标定。

## 2. 像素坐标协议 `T`

```text
T,seq,zone_id,cx,cy,img_w,img_h,score,light,beep
```

示例：

```text
T,43,1,420,260,640,360,0.811,1,2
```

ESP32 将坐标归一化后转换为舵机角度。该模式方便兼容，但精确标定能力弱于 `A`。

## 3. 旧协议 `@T`

```text
@T,seq,valid,xn,yn,danger,alarm
```

`xn/yn` 范围为 0–1000，只用于兼容旧固件或旧主机程序。

## 4. 控制命令

```text
C       # 回中并停止报警
L,seq   # 目标丢失/停止报警
```

## 5. ACK

固件返回以 `ACK:` 开头的状态行，可能包含序号、目标角度、滤波坐标和报警等级。
未知或字段不足的命令返回 `ERR`。

## 6. 协议选择

在 `configs/runtime.yaml` 设置：

```yaml
esp32_protocol: A
```

配置允许 `A`、`T`、`AT`。`AT` 表示发送旧 `@T` 格式。

## 7. 安全约束

- 序号不能替代完整的通信完整性校验；
- 当前协议未包含认证、加密或 CRC，不应跨不受信任链路传输；
- `light` 字段存在于协议中，但公开固件尚未绑定已验证灯光 GPIO；
- 串口丢失、解析错误或超时必须进入停止报警/目标丢失状态；
- 手动调试命令只能在人员撤离、执行器限位确认后使用。
