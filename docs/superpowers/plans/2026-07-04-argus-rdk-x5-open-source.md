# ARGUS RDK X5 Open-Source Publication Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use executing-plans to implement this plan inline. The user requested direct execution without a separate planning checkpoint.

**Goal:** Turn the final ARGUS archive into a readable, reproducible public repository and publish source plus a model-free `v1.0.0` release.

**Architecture:** Preserve the archive's online and offline Python applications with minimal runtime-safety changes, move the shared ESP32 firmware to one canonical directory, and document the external model contract. Hardware, servo, buzzer, and LLM integrations remain disabled by default.

**Tech Stack:** Python 3.10, FastAPI, OpenCV, NumPy, RDK X5 `hobot_dnn`, ESP32 Arduino, JSON/YAML, GitHub Actions.

## Global Constraints

- Public repository name: `argus-rdk-x5`.
- Public repository description must match the supplied text exactly.
- Keep online Web, offline preview, ESP32 firmware, frontend, configuration, scripts, and deployment documentation.
- Do not commit datasets, media, caches, credentials, private configuration, archives, or model binaries.
- Model interface: 640×640 NV12, 3 classes, strides 8/16/32, DFL reg 16, six alternating classification/regression outputs.
- Default host is `127.0.0.1`; all hardware, servo, buzzer, and LLM switches are false.
- Release tag and title: `v1.0.0`, `ARGUS v1.0.0 — Public Research Prototype`.
