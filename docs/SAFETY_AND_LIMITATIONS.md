# 安全边界与限制

ARGUS 是研究与工程原型，不能替代工业安全认证系统、急停装置、机械限位或人工安全管理。

## 默认安全状态

- Web 仅监听 `127.0.0.1`；
- `hardware_enabled: false`；
- `servo_enabled: false`；
- `buzzer_enabled: false`；
- `light_enabled: false`；
- `llm_bridge_enabled: false`；
- 模型缺失时明确报错；
- CI 不连接任何真实硬件。

## 视觉限制

- 遮挡、背光、运动模糊和摄像头污染会降低检测质量；
- helmet/vest 与 person 的关联基于几何启发式，不保证遮挡场景正确；
- 跨帧平滑会引入延迟；
- 危险区域是二维图像区域，不能表达真实三维安全距离；
- 模型精度、数据分布和量化损失需要现场独立验证。

## 决策限制

- 风险分数是工程启发式，不对应安全完整性等级；
- 多目标仲裁一次只选择一个执行方向；
- 目标丢失保持用于减少抖动，不代表目标仍然存在；
- LLM 输出不参与核心安全决策，默认关闭；
- Web API 的手动控制不应对公网开放。

## 执行器限制

- 软件角度限制不能替代机械限位；
- 舵机、蜂鸣器和灯光需要独立电气设计与失效分析；
- 当前公开固件没有绑定已验证灯光 GPIO；
- 串口没有认证、加密和 CRC；
- ESP32 超时回退不能替代硬件急停。

## 模型和许可

公开仓库不包含训练数据、校准图像或模型二进制。模型使用者必须确认：

- 数据采集和标注合法；
- 模型允许再分发与部署；
- Ultralytics、D-Robotics 和其他工具链许可得到遵守；
- 模型在目标现场经过性能、偏差和失效测试。

## 部署前检查

- 人员撤离执行器工作范围；
- 电源、共地、驱动与限位正确；
- 摄像头固定牢靠；
- 危险区域和舵机映射重新标定；
- 无硬件模式运行稳定；
- 串口丢失和断电回退已测试；
- 现场仍有独立急停、隔离和人工安全流程。
